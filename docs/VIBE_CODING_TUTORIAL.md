# 바이브 코딩 튜토리얼: 비개발자를 위한 AI 프로젝트 만들기

> **바이브 코딩(Vibe Coding)** 이란?
> 코드를 직접 작성하지 않고, AI에게 자연어로 원하는 것을 설명하여 소프트웨어를 만드는 방식입니다.
> 이 튜토리얼에서는 **Claude Code**를 사용하여 실제 웹 애플리케이션을 만드는 전체 과정을 보여드립니다.

---

## 사전 준비

### 1. Claude Code 설치

터미널(Mac의 경우 Terminal 앱)을 열고 아래 명령어를 입력합니다:

```
npm install -g @anthropic-ai/claude-code
```

### 2. Claude Code 실행

프로젝트를 만들 폴더로 이동한 후 Claude Code를 실행합니다:

```
mkdir my-project
cd my-project
claude
```

이제 Claude Code와 대화할 수 있는 상태가 됩니다.

---

## Step 1: 아이디어를 구체화하기 (심층 인터뷰)

바이브 코딩의 핵심은 **"무엇을 만들고 싶은지 명확하게 설명하는 것"** 입니다.

### 프롬프트 예시

```
온라인 강의 시스템을 만들고 싶어.
구체적으로 이 프로젝트의 PRD를 만들기 위한 심층 인터뷰를 진행해줘.
```

### 무슨 일이 일어나나요?

Claude Code가 여러분에게 **질문을 하나씩** 던집니다:

- "핵심 학습 방식은 무엇인가요?"
- "대상 사용자는 누구인가요?"
- "어떤 기능이 필요한가요?"

여러분은 **한국어로 자연스럽게 대답**하면 됩니다:

- "강의는 없고 챌린지만 있어"
- "모두 대상이야"
- "웹 플랫폼도 필요해"

Claude Code는 매 답변 후 **명확도 점수**를 보여주며, 충분히 명확해지면 자동으로 PRD(제품 요구사항 문서)를 생성합니다.

### 핵심 팁

- 짧고 명확하게 대답하세요
- 모르는 것은 "알아서 정해줘"라고 해도 됩니다
- 선택지가 나오면 가장 적합한 것을 골라주세요

---

## Step 2: GitHub 저장소 만들기

코드를 저장하고 관리할 공간을 만듭니다.

### 프롬프트 예시

```
이 프로젝트를 위한 리포지토리를 생성하고 깃헙 my-org에 퍼블릭으로 등록해줘.
```

### 무슨 일이 일어나나요?

Claude Code가 자동으로:
1. Git 저장소를 초기화합니다
2. README 파일을 생성합니다
3. GitHub에 저장소를 등록합니다

여러분은 GitHub 계정만 연결되어 있으면 됩니다.

---

## Step 3: 목업(화면 시안) 만들기

실제 동작하는 화면을 만들어봅니다. 이 단계에서는 디자인만 구현하고, 실제 데이터 연동은 나중에 합니다.

### 프롬프트 예시

```
PRD 기반으로 웹 플랫폼의 UI 목업을 구현해줘.
Next.js + Tailwind CSS로 다음 페이지들을 만들어줘:
1) 랜딩 페이지
2) 로그인/회원가입
3) 챌린지 목록
4) 챌린지 상세 페이지
5) 수강생 대시보드
6) 관리자 페이지
실제 API 연동 없이 하드코딩된 샘플 데이터를 사용해줘.
```

### 무슨 일이 일어나나요?

Claude Code가:
1. 프로젝트를 자동으로 생성합니다 (Next.js 설치)
2. 샘플 데이터를 만듭니다
3. 각 페이지를 디자인하고 코드를 작성합니다
4. 빌드하여 에러가 없는지 확인합니다

### 결과 확인하기

```
화면을 브라우저에서 확인하고 싶어.
```

Claude Code가 개발 서버를 실행하고, 브라우저에서 각 페이지를 스크린샷으로 보여줍니다.

### 수정 요청하기

화면이 마음에 들지 않으면 자연어로 수정을 요청하세요:

```
랜딩 페이지의 배경색을 좀 더 밝게 바꿔줘.
```

```
챌린지 카드에 진행률 바를 추가해줘.
```

```
모바일에서 메뉴가 잘 안보여. 더 크게 만들어줘.
```

---

## Step 4: 개발 계획 만들기

실제 개발을 위한 상세 계획을 수립합니다.

### 프롬프트 예시

```
이 프로젝트를 개발자들이 개발할 수 있도록 계획문서를 작성하고
계획 문서의 항목들을 깃헙 이슈에 등록해줘.
깃헙 이슈에는 작업 배경, 작업 내용, 인수 조건, 다른 작업과의 의존성이 포함되어야 해.
```

### 무슨 일이 일어나나요?

Claude Code가:
1. 개발 단계를 Phase별로 나눕니다 (기반 구축 → 인증 → 핵심 기능 → 배포)
2. 각 작업 항목에 대한 상세 설명을 작성합니다
3. GitHub에 이슈(작업 티켓)를 자동 등록합니다
4. 작업 간 의존 관계를 명시합니다

---

## Step 5: 저장하기 (커밋 & 푸시)

작업 내용을 GitHub에 저장합니다.

### 프롬프트 예시

```
작업 내용 모두 커밋하고 푸시해.
```

이 한 줄이면 Claude Code가 알아서:
1. 변경된 파일을 확인합니다
2. 적절한 커밋 메시지를 작성합니다
3. GitHub에 업로드합니다

---

## 바이브 코딩 핵심 원칙

### 1. 원하는 "결과"를 설명하세요 (방법이 아닌)

```
(좋은 예) "수강생이 기술스택별로 챌린지를 필터링할 수 있게 해줘"
(나쁜 예) "React의 useState 훅을 사용해서 필터 상태를 관리하고..."
```

### 2. 한 번에 하나씩 요청하세요

```
(좋은 예) "먼저 로그인 페이지를 만들어줘"
(나쁜 예) "로그인, 회원가입, 비밀번호 찾기, 소셜 로그인, 2FA를 전부 한번에 만들어줘"
```

### 3. 결과를 확인하고 피드백하세요

```
"로그인 페이지가 만들어졌어? 보여줘."
"입력 필드가 너무 작아. 더 크게 만들어줘."
"비밀번호 입력할 때 눈 아이콘으로 표시/숨기기 할 수 있게 해줘."
```

### 4. 모르는 건 물어보세요

```
"이 프로젝트에서 데이터베이스는 뭘 쓰는 게 좋아?"
"인증은 어떤 방식으로 구현하는 게 좋아?"
"AWS에서 가장 저렴하게 배포하려면 어떻게 해야 해?"
```

### 5. "알아서 해줘"도 괜찮습니다

```
"프론트엔드 프레임워크는 알아서 정해줘."
"프로젝트 구조는 베스트 프랙티스로 알아서 잡아줘."
```

## 자주 쓰는 프롬프트 모음

### 프로젝트 시작

| 상황 | 프롬프트 |
|------|---------|
| 새 프로젝트 시작 | "OOO 시스템을 만들고 싶어. 심층 인터뷰를 진행해줘." |
| 저장소 생성 | "깃헙에 리포지토리를 만들고 등록해줘." |
| 기존 프로젝트 분석 | "이 프로젝트의 구조를 분석해줘." |

### 화면 만들기

| 상황 | 프롬프트 |
|------|---------|
| 목업 생성 | "PRD 기반으로 UI 목업을 만들어줘." |
| 디자인 수정 | "이 페이지의 색상을 더 밝게 바꿔줘." |
| 기능 추가 | "검색 기능을 추가해줘." |
| 반응형 확인 | "모바일에서 어떻게 보이는지 확인해줘." |

### 개발 관리

| 상황 | 프롬프트 |
|------|---------|
| 계획 수립 | "개발 계획을 세우고 이슈로 등록해줘." |
| 진행 확인 | "현재 프로젝트 상태를 요약해줘." |
| 버그 수정 | "이 페이지에서 OOO가 안돼. 고쳐줘." |

### 저장 및 배포

| 상황 | 프롬프트 |
|------|---------|
| 저장 | "작업 내용 커밋하고 푸시해." |
| PR 생성 | "PR을 만들어줘." |
| 배포 | "이 프로젝트를 Vercel에 배포해줘." |

## 마무리

바이브 코딩은 **코드를 몰라도** 소프트웨어를 만들 수 있게 해주는 혁신적인 방법입니다. 핵심은:

1. **무엇을 만들고 싶은지 명확하게 설명하기**
2. **결과를 확인하고 피드백하기**
3. **모르는 것은 AI에게 물어보기**

이 세 가지만 기억하면, 누구나 자신만의 소프트웨어를 만들 수 있습니다.
