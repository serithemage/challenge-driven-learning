## 1. Product Vision

"강의 없이 챌린지만으로 학습한다."

Challenge Driven Learning(CDL)은 수강생이 코드를 직접 작성하거나 볼 필요 없이, Claude Code를 활용하여 원하는 결과물을 얻는 방법을 학습하는 교육 플랫폼이다. 학습의 핵심은 **최종 결과물을 명확하게 정의하는 능력**을 기르는 것이다.

## 2. Target Users

| 사용자 유형 | 설명 | 주요 니즈 |
|------------|------|----------|
| **수강생 (개발자)** | AI 활용 능력을 높여 생산성을 극대화하려는 개발자 | 실무 기술스택에 맞는 챌린지, 자동 평가 피드백 |
| **수강생 (비개발자)** | 기획자, 디자이너, 마케터 등 코딩 경험 없이 AI로 제품을 만들고자 하는 직군 | 코드 없이 결과물을 정의하고 얻는 방법 학습 |
| **관리자** | 한빛미디어 교육 콘텐츠 담당자 | 챌린지 등록/관리, 수강생 진행 모니터링 |

## 3. Core Concept

### 3.1 챌린지 기반 학습

- 전통적인 강의(영상, 텍스트)가 없음
- 수강생은 **리얼월드 앱 구현체**를 소재로 프로젝트형 챌린지를 수행
- 업무에서 사용하는 기술스택과 가장 유사한 구현체를 선택

### 3.2 챌린지 진행 단계

```
1. 결과물 정의 (Outcome Definition)
   └─ 수강생이 최종 결과물을 명확하게 정의

2. 문서화 (Documentation)
   └─ Claude Code를 활용해 프로젝트 문서 생성

3. 테스트 자동화 (Test Automation)
   └─ Claude Code를 활용해 테스트 코드 생성 및 자동화

4. 훅 구현 (Hooks)
   └─ Claude Code 훅을 설계하고 구현

5. 스킬 구현 (Skills)
   └─ Claude Code 스킬(.md)을 설계하고 구현
```

### 3.3 티칭모드 플러그인

- **형태:** Claude Code 슬래시 커맨드(스킬) `.md` 파일
- **역할:** 챌린지 진행 시 학습 가이드
- **동작 방식:**
  - 수강생이 결과물 정의에 집중하도록 안내
  - 단계별 진행을 추적
  - 코드를 직접 보여주지 않고 결과물 관점에서 가이드

### 3.4 자동 평가 시스템

| 평가 계층 | 담당 | 평가 내용 |
|----------|------|----------|
| **CI/CD (GitHub Actions)** | 자동 | 테스트 통과, 린트, 커버리지, 빌드 성공 |
| **Claude Code 리뷰** | 자동 | 산출물 품질 검토, 개선 피드백 제공 |

## 7. MVP Scope

### In Scope (MVP)
- 이메일 기반 회원가입/로그인
- 챌린지 CRUD (관리자)
- 챌린지 목록 조회 및 시작 (수강생)
- 티칭모드 스킬 파일 제공
- CI/CD + Claude Code 자동 평가
- 진행 상황 대시보드
- 수강생 모니터링 (관리자)

### Out of Scope (MVP)
- 결제 시스템
- 영상 강의
- 한빛미디어 기존 플랫폼 연동
- 모바일 네이티브 앱
- 소셜 로그인
- 수강생 간 소통 기능 (포럼, 댓글 등)
