# 비개발자를 위한 Git 가이드 (Claude Code 프롬프트 중심)

> Git은 코드의 **변경 이력을 관리**하는 도구입니다.
> 마치 구글 독스의 "버전 기록"처럼, 언제 누가 무엇을 변경했는지 추적합니다.
> Claude Code를 사용하면 Git 명령어를 몰라도 됩니다 — **한국어로 말하면** 됩니다.

---

## Git 핵심 개념 (5가지만 알면 됩니다)

### 1. 저장소 (Repository)

프로젝트의 모든 파일과 변경 이력이 저장되는 공간입니다.

- **로컬 저장소:** 내 컴퓨터에 있는 프로젝트 폴더
- **원격 저장소:** GitHub에 있는 프로젝트 (클라우드 백업이라고 생각하세요)

### 2. 커밋 (Commit)

"저장" 버튼과 같습니다. 현재 작업 내용을 기록합니다.
각 커밋에는 **무엇을 왜 변경했는지** 메시지가 포함됩니다.

### 3. 푸시 (Push)

내 컴퓨터의 커밋을 GitHub(원격 저장소)에 **업로드**하는 것입니다.
다른 팀원들이 내 작업을 볼 수 있게 됩니다.

### 4. 풀 (Pull)

GitHub에서 다른 사람이 올린 변경사항을 내 컴퓨터로 **다운로드**하는 것입니다.

### 5. 브랜치 (Branch)

원본을 건드리지 않고 **별도의 작업 공간**을 만드는 것입니다.
완성되면 원본에 합칩니다(머지).

## 상황별 Claude Code 프롬프트

### 처음 시작할 때

#### 새 저장소 만들기

```
이 프로젝트를 위한 깃 저장소를 만들고 깃헙에 등록해줘.
```

Claude Code가 하는 일:
- `git init` (저장소 초기화)
- README 파일 생성
- GitHub에 저장소 생성
- 첫 커밋 및 푸시

#### 기존 저장소 가져오기

```
깃헙의 https://github.com/someone/project 를 클론해줘.
```

---

### 작업 저장하기

#### 기본 저장 (커밋 + 푸시)

```
작업 내용 커밋하고 푸시해.
```

이것이 가장 자주 쓰는 프롬프트입니다. Claude Code가 알아서:
1. 변경된 파일을 확인합니다
2. 적절한 커밋 메시지를 작성합니다
3. GitHub에 업로드합니다

#### 특정 메시지로 저장하기

```
"로그인 페이지 디자인 완성"이라는 메시지로 커밋하고 푸시해.
```

#### 일부 파일만 저장하기

```
프론트엔드 파일만 커밋해줘. docs 폴더는 아직 작업 중이야.
```

---

### 현재 상태 확인하기

#### 변경 사항 확인

```
현재 어떤 파일이 변경되었는지 알려줘.
```

Claude Code가 `git status`를 실행하고 결과를 한국어로 설명해줍니다:
- 새로 생성된 파일
- 수정된 파일
- 삭제된 파일

#### 변경 내용 상세 확인

```
마지막 커밋 이후로 뭐가 변경됐는지 보여줘.
```

#### 커밋 이력 확인

```
최근 커밋 내역을 보여줘.
```

---

### 브랜치 작업하기

#### 새 브랜치에서 작업 시작

```
'feature/login' 브랜치를 만들고 거기서 작업할게.
```

#### 현재 브랜치 확인

```
지금 어떤 브랜치에서 작업하고 있어?
```

#### 메인 브랜치로 돌아가기

```
main 브랜치로 돌아가줘.
```

#### 작업 완료 후 합치기

```
이 브랜치의 작업이 끝났어. PR을 만들어줘.
```

---

### Pull Request (PR) 작업하기

PR은 "내 작업을 검토해주세요"라는 요청입니다.

#### PR 생성

```
현재 브랜치로 PR을 만들어줘.
```

Claude Code가:
1. 변경 내용을 분석합니다
2. PR 제목과 설명을 자동 작성합니다
3. GitHub에 PR을 생성합니다

#### PR 확인

```
이 저장소에 열린 PR 목록을 보여줘.
```

---

### 다른 사람의 변경사항 가져오기

#### 최신 코드 받기

```
원격 저장소에서 최신 코드를 받아줘.
```

#### 충돌이 발생했을 때

```
머지 충돌이 발생했어. 해결해줘.
```

Claude Code가 충돌 부분을 분석하고, 어떻게 해결할지 물어봅니다.

---

### 실수했을 때

#### 마지막 변경 취소

```
방금 수정한 파일을 원래대로 되돌려줘.
```

#### 특정 파일 되돌리기

```
index.html 파일을 마지막 커밋 상태로 되돌려줘.
```

#### 커밋 전 상태 확인

```
커밋하기 전에 변경된 내용을 먼저 보여줘.
```

---

### GitHub 이슈 관리

#### 이슈 확인

```
깃헙에 등록된 이슈 목록을 보여줘.
```

#### 이슈 생성

```
"로그인 시 비밀번호 검증 추가" 이슈를 만들어줘.
```

#### 이슈 닫기

```
3번 이슈를 닫아줘.
```

## 일상적인 작업 흐름

매일 작업할 때 이 순서를 따르세요:

### 1. 작업 시작

```
최신 코드를 받아줘.
```

### 2. 작업 진행

Claude Code와 대화하며 기능을 구현합니다.

### 3. 작업 저장

```
작업 내용 커밋하고 푸시해.
```

### 4. (선택) PR 생성

팀으로 작업할 때:

```
PR을 만들어줘.
```

## 자주 하는 실수와 해결법

### "푸시가 안돼요"

```
푸시가 안돼. 원인을 확인하고 해결해줘.
```

보통 원격 저장소에 내가 없는 변경사항이 있어서 발생합니다.
Claude Code가 자동으로 최신 코드를 받고 다시 푸시합니다.

### "파일이 사라졌어요"

```
어제까지 있던 파일이 없어졌어. 복구해줘.
```

Git은 모든 이력을 저장하므로, 삭제된 파일도 복구할 수 있습니다.

### "잘못된 브랜치에서 작업했어요"

```
이 변경사항을 다른 브랜치로 옮겨줘.
```

### ".env 같은 민감한 파일을 실수로 올렸어요"

```
.env 파일을 깃 추적에서 제외해줘. 이미 올라간 것도 제거해줘.
```

## 핵심 정리

| 하고 싶은 것 | Claude Code 프롬프트 |
|-------------|---------------------|
| 저장소 만들기 | "깃 저장소 만들고 깃헙에 등록해줘" |
| 작업 저장 | "커밋하고 푸시해" |
| 상태 확인 | "변경된 파일 알려줘" |
| 이력 확인 | "최근 커밋 내역 보여줘" |
| 최신 코드 받기 | "원격에서 최신 코드 받아줘" |
| 브랜치 만들기 | "'feature/이름' 브랜치 만들어줘" |
| PR 만들기 | "PR 만들어줘" |
| 되돌리기 | "마지막 변경 되돌려줘" |
| 문제 해결 | "OOO가 안돼. 해결해줘" |

**기억하세요:** Git 명령어를 외울 필요가 없습니다. 하고 싶은 것을 **한국어로 말하면** Claude Code가 알아서 처리합니다.
